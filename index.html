How to pubilish your Hexo blog hosted on Github with your ipad gracefully?

Posted on 2019-12-25 In HowTo

In the last days, I was on a vacation. I wanted to push my news to my own blog, but the only thing I had with me is my ipad Pro. So I asked myself, is there a way to publish something with ipad “gracefully”?

The answer is: yes, of course!

I assume that, you already know how to generate your blog and deploy it on github or your own server with Hexo. If you still have questions about that, you can refer to my last article about it.

My first thought is to get remote access to your server and edit your post directly on it. I’m using Promp2 as my tool to work on a remote server and its performance is satisfying sofar.

However, we are talking about how to edit your post gracefully. In this way, you have to use the command lines and do a lot of work to get your article published, including edit the post, generate the static files and deploy them on git or your own server. Especially if you want to edit your post with many pictures, you have to upload the pictures and rename them first. To add them in the post is not very convenient. If you need to post news very often, this extra work will take a lot of time.

How can you make the process easier, so that you can focus on your writing?

I made another git repository with my hexo source files. With a tool from app store called Working Copy, you can get access to your repo and do the normal work such as pull, commit and push easily.

With this tool, you can add your post files in this application and push them to your remote repo.

You have to setup a webhook, which allows you to to trigger a bash script on your server after every push. This script will clean the hexo static files, generate the new public files and deploy them to the other git repository, which you use to host your blog or deploy them to your own server. After that, this script will also push the whole scource file to remote again, so that it is sychronized among your server, your ipad and github.

However, it still doesn’t solve the problem with inserting pictures. You can upload your pictures on a image host. And use the url you get from that in your posts. But normally there are a lot restrictions about the size and number of your pictures you are allowed to upload. Therefore I prefer to host the pictures on my own site.

You can use Working Copy to upload the pictures manuelly to your repo and insert them in your markdown file. There is another way, which make it much easier and faster. There are a lot of markdown editors in app store. I’m using an app called noto. Using this tool, you can focus on editing your blog and use the functions such as insert a picture or formula just as in microsoft word. After that, you can export your file in formats such as pdf, html or markdown.

If you export it as markdown, you will get a folder with .md file and all the pictures you used in your file. And the images in the md file are inserted directly with their names. But hexo requires you to add the images in the folder …/source/images/ and your md files in the folder …/source/_posts. And you have to use the images with the url such as /images/….jpg.

After export your markdown folder from noto, I move the folder to a new folder named update in the github repository. On the server side, you have to add an extra script with webhook, which will edit the md file and move it to the _posts folder. And then make a new folder with the same name as your post in the images folder. After that, it will move all the pictures to this folder.

There are some points, you have to pay attention to, before you fall into the traps. In the name of this md files, there must be no space or special characters. The md file from noto is automatically named as content.md. And you cannot change the name automatically in noto, or at least I haven’t found the function yet. So I implement this function in my script with webhook. It will read out the content in my md file in second row after “title: “ and rename my md file and the folder for images.

To summary, with all the work mentioned above, your final work process will be looking like this:

1, You edit your post in noto and export it in the folder update. In Working Copy, there is an option to make the folder update automatically sychronized.
2, You have to push the new commits to your github repo.

After this two steps, your server and github will do the rest of the work. After few miniutes, your hexo blog will be updated.
